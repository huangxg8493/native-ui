# 用户密码重置功能设计

## 背景

将 user.html 中的"修改密码"功能改为"重置密码"功能。管理员可为任意用户重置密码，无需验证旧密码，直接重置为默认密码。

## 接口

- **URL**: `POST /api/users/{userId}/password/reset`
- **认证**: JWT Token
- **请求体**: `{ "newPassword": "123456" }`

## 功能设计

### 用户操作流程

1. 管理员在用户管理表格中点击"重置密码"按钮
2. 弹出确认框："确定要重置该用户密码吗？重置后密码为 123456"
3. 确认后调用重置接口
4. 成功后 Toast 提示"重置成功，密码已重置为 123456"

### UI 改动

#### HTML（user.html）

- 删除 `#passwordModal`（修改密码弹窗）
- 按钮文案从"修改密码"改为"重置密码"

#### JS（user.js）

- 删除 `passwordForm` 相关逻辑
- `changePwdBtn` 点击事件改为：
  - 弹出 `confirm()` 确认框
  - 确认后直接调用 `POST /api/users/{userId}/password/reset`
  - body 固定 `{ newPassword: "123456" }`
  - 成功提示"重置成功，密码已重置为 123456"

## 错误处理

| 错误码 | 处理 |
|--------|------|
| 000000 | Toast 成功提示 |
| 101006 | Toast 提示"密码格式错误" |
| 101008 | Toast 提示"用户不存在" |
| 网络错误 | Toast 提示"网络错误" |
