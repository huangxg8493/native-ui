# 用户管理页面设计

**日期**: 2026-04-28
**接口文档**: `http/interface.md`

---

## 1. 概述

新建用户管理页面 `html/client/user.html`，独立完整页面风格（与 `address.html` 一致），实现用户 CRUD 和角色分配功能。

## 3. 搜索栏

| 字段 | 类型 | 默认值 |
|------|------|--------|
| 手机号 | input text | 空 |
| 状态 | select | 全部 |
| | | options: 全部 / 正常 / 禁用 |

- 查询按钮：触发列表刷新
- 新增用户按钮：打开新增弹窗

---

## 4. 数据表格

**列定义：**

| 列名 | 来源字段 | 说明 |
|------|----------|------|
| 序号 | 索引 | 从1开始 |
| 手机号 | phone | |
| 用户名 | userName | 可为空，显示 "-" |
| 邮箱 | email | 可为空 |
| 省市 | province + city | 如 "广东省/深圳市" |
| 状态 | status | Y=正常(绿色) N=禁用(红色) |
| 创建时间 | createTime | 格式: YYYY-MM-DD HH:mm |
| 操作 | - | 编辑/删除/分配角色 三个按钮 |

**状态显示：**
- `status = 'Y'` → 绿色文字 "正常"
- `status = 'N'` → 红色文字 "禁用"

**操作按钮：**
- `编辑` → 蓝色按钮，打开编辑弹窗
- `删除` → 红色按钮，弹出确认后调用删除接口
- `分配角色` → 蓝色按钮，打开角色分配弹窗

---

## 5. 弹窗设计

### 5.1 新增/编辑用户弹窗

**字段（新增）：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 手机号 | input | 是 | 11位手机号 |
| 密码 | input (password) | 是 | 最少6位 |
| 状态 | select | 否 | 默认 Y |

**字段（编辑）：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用户ID | hidden | - | 用于提交 |
| 手机号 | input | 是 | 不可编辑 |
| 密码 | input (password) | 否 | 空则不修改 |
| 状态 | select | 否 | |

**按钮：** 取消 / 确定

### 5.2 分配角色弹窗

- 标题：根据手机号显示如 "为 13900000001 分配角色"
- 内容：角色列表（从 `localStorage.roles` 获取），复选框形式
- 接口调用：`POST /api/users/{userId}/roles/assign`，body 传 `[roleId1, roleId2, ...]`
- 成功后刷新列表

---

## 6. 分页

- 每页条数选项：5 / 10 / 20 / 50，默认 10
- 上一页/下一页按钮
- 接口参数：`pageNum`, `pageSize`
- 搜索时重置 pageNum = 1

---

## 7. HTTP 请求

**查询列表：** `POST /api/users/query`
```json
{
    "phone": "13900000001",
    "status": "Y",
    "pageNum": 1,
    "pageSize": 10
}
```

**新增用户：** `POST /api/users/create`
```json
{
    "phone": "13900000002",
    "password": "password123",
    "status": "Y"
}
```

**更新用户：** `POST /api/users/update`
```json
{
    "userId": 1234567890,
    "password": "newpassword123",
    "status": "Y"
}
```

**删除用户：** `POST /api/users/delete` + query `?userId=xxx`

**分配角色：** `POST /api/users/{userId}/roles/assign`
```json
[1, 2, 3]
```

## 9. 交互流程

1. 进入页面 → 自动查询第一页列表
2. 输入手机号/选择状态 → 点查询 → 刷新列表
3. 点新增 → 弹出空白表单 → 填写 → 提交成功 → 刷新列表
4. 点编辑 → 弹出预填表单 → 修改 → 提交成功 → 刷新列表
5. 点删除 → 弹出确认 → 确认后删除 → 刷新列表
6. 点分配角色 → 弹出角色勾选 → 勾选后提交 → 刷新列表
7. 分页切换 → 请求对应页数据

---

## 10. Toast 提示

- 操作成功：绿色 toast，2秒后消失
- 操作失败：红色 toast，显示后端返回 message
