# 用户管理 - 修改密码功能独立化设计

## 背景

当前编辑弹窗中包含密码字段，用于修改密码。为明确职责，将密码修改功能独立出来：
- **编辑按钮**：仅编辑用户基本信息（用户名、邮箱、省市区、状态），不再包含密码修改
- **修改密码按钮**：独立的修改密码流程

## 接口

### 更新用户密码

- **接口 URL**: `POST /api/users/update/pwd`
- **Content-Type**: `application/json`

**请求体**:
```json
{
    "userId": 1234567890,
    "phone": "13900000002",
    "password": "原密码",
    "newPassword": "新密码"
}
```

**成功响应**: `{"code": "000000", "message": "成功", "data": null}`

## 功能设计

### 1. 操作列按钮

表格操作列调整为4个按钮：`编辑` | `修改密码` | `删除` | `分配角色`

### 2. 编辑弹窗调整

移除密码字段，保留以下字段（创建时间、更新时间只读）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 手机号 | input | 只读 | 不可修改 |
| 用户名 | input | 否 | |
| 邮箱 | input | 否 | |
| 省市区 | input | 否 | 取 city 字段 |
| 状态 | select | 是 | Y=正常，N=禁用 |
| 创建时间 | input | 只读 | |
| 更新时间 | input | 只读 | |

### 3. 修改密码弹窗（新建）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 原密码 | password | 是 | |
| 修改密码 | password | 是 | 最少6位 |
| 确认密码 | password | 是 | 必须与修改密码一致 |

**交互流程**：
1. 点击"修改密码"按钮，弹出密码修改弹窗
2. 用户输入原密码、新密码、确认密码
3. 前端校验：确认密码与新密码一致，新密码最少6位
4. 提交 `/api/users/update/pwd`
5. 成功：提示"密码修改成功，即将跳转到登录页"，2秒后自动跳转登录页
6. 失败：显示错误提示

### 4. 密码修改成功后跳转登录页

调用 `window.location.href = 'login.html'` 或跳转到项目登录页。

## 涉及的代码文件

| 文件 | 改动 |
|------|------|
| `html/sys/user.html` | 编辑弹窗移除密码字段；新增修改密码弹窗 HTML |
| `assets/script/user.js` | 编辑逻辑移除密码处理；新增修改密码弹窗打开/提交/关闭逻辑 |
| `styles/user.css` | 如有需要，新增修改密码弹窗样式 |

## 实现步骤

1. **HTML**：编辑弹窗移除密码行；新增修改密码弹窗结构
2. **JS**：编辑回填逻辑移除密码字段；新增 modifyPassword 函数绑定修改密码按钮事件；表单提交调用 `/api/users/update/pwd`
3. **JS**：密码修改成功后 2 秒跳转到登录页
